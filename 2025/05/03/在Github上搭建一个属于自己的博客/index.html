二、Hexo环境配置及安装

git

git官网：Git - Downloads

Node.js

1.下载安装

Node.js官网：https://nodejs.org/zh-cn/download/

2.修改包路径

默认情况下，包存在C盘，我们可以通过设置改变包存放路径。

在安装目录下创建两个文件夹：

node_global node_cache

在cmd中执行：

npm config set prefix "D:\nodejs\node_global"

npm config set cache "D:\nodejs\node_cache"

现在再通过npm root -g查看包存放路径，已经不再是C盘了

3.修改环境变量

在上面用户变量->双击Path->“新建”D:\nodejs\node_global`

在下面系统变量->“新建”->变量名为NODE_PATH变量值为D:\nodejs\node_modules（你的node_modules目录地址）

Hexo

Hexo官网：Hexo

1.命令行安装

以管理员身份运行cmd，cd到你想安装的目录下，执行如下命令操作：npm install -g hexo-cli

完事通过hexo -v来检测是否安装成功

补：

1.cd是cmd里访问文件夹的命令，例如我想访问C:\System，就是cd c:\Program Files

2.管理员模式下cmd访问其他盘符(以D:为例)的方法：输入cd /d d:即可管理员下访问D盘

2.初始化

以管理员身份运行cmd，且cd到在hexo-blog文件夹下，执行如下命令：

hexo init hexo-blog

cd hexo-blog

npm install

3.本地启动

以管理员身份运行cmd，且cd到在hexo-blog文件夹下，执行如下命令：

hexo g

hexo server

可以访问了！

网址为：http://localhost:4000

补充：日后如何发表博客？

当时跟着步骤做的，就安装到C盘了！经过验证可以直接将C盘的hexo-blog移动到其他盘

上传博客的正确步骤是：

  1. 使用hexo new “文章标题”创建文章文件。
  2. 将图片存放在source目录下的合适文件夹（如source/images），并在文章中引用。
  3. 编写文章内容。
  4. 使用hexo g生成静态文件，使用hexo d部署到 GitHub。

删除博客的正确步骤是：

  1. 找到source/_posts文件夹，删除要删除的文章对应的.md文件和相关图片。
  2. 使用hexo clean清理缓存。
  3. 使用hexo g重新生成静态文件，使用hexo d部署到 GitHub。

常见问题

​ 这些报错都是我在做的时候出现的问题，如果你是按着本篇文章顺序做下来的，那应该很顺利。以下报错是给一些在做博客时苦苦挣扎的朋友们提供一个字典式的查询方案。

1.管理员模式下的cmd无法切换盘符

有童鞋反应说，管理员模式打开cmd后，无法cd到其他盘符。

原因：在管理员权限打开cmd时，使用 cd 路径 的方式切换路径时，只能是切换到当前盘符下的目录，而不能切换到其他盘符。

解决方法：在 cd 路径 之间添加一个 /d 或者 /D 来进行切换，这样就可以切换到其他盘符的目录下。

如： cd /d D:\Util\MySql\mysql-5.6.44-winx64\bin

2.在npm安装Hexo时 出现的npm error code EPERM…

npm error code EPERM

npm error syscall mkdir

npm error path D:\nodejs\node_cache_cacache

npm error errno EPERM

原因：没有权限创建文件夹

解决方法：使用管理员身份运行CMD

3.LeanClouds配置后不显示文章阅读量

这个问题也困扰了我许久，后来终于发现views下的sourse数据来源不能填leancloud（即便我们用的就是leancloud），应该填busuanzi，就可以现实了

见下图：将hexo-blog/themes/fluid/_config.yml中views里的source改为busuanzi

应该等部署到github上就OK了
